* Hoogle for C#

A type-signature search tool for C# / .NET, modeled after Haskell's [[https://hoogle.haskell.org/][Hoogle]].
Search the BCL, your project's NuGet dependencies, and your own source code by
type signature (e.g. =IEnumerable<T> -> Func<T,bool> -> IEnumerable<T>=) or by
case-insensitive substring on the method's full name.

** Requirements

- .NET 8.0 SDK

** Build

#+begin_src sh
dotnet build
#+end_src

** Usage

#+begin_example
Usage: csharp-hoogle [options] <query>

  <query>                       If it contains "->", matched as a type signature
                                (e.g. `IEnumerable<T> -> Func<T,bool> -> IEnumerable<T>`).
                                Otherwise, case-insensitive substring on the full name.
  --rebuild                     Rebuild the on-disk cache before searching.
  --list-assemblies             List assemblies in the cache with method counts.
  --json                        Emit results as JSON on stdout (for piping into tools).
  --polymorphic                 Include matches where a concrete query type binds a method's
                                declared generic parameter (e.g. Func.Invoke for Int32 -> Int32 -> Int32).
  --project <path>              Pin context to a specific .sln/.slnx/.csproj file.
  --target-framework <tfm>, -f  Pin TFM (e.g. net8.0); overrides on-disk detection.
  --max-results <int>           Set maximum number of results (default: no limit)
  --help                        Show this message.
#+end_example

Without overrides, =csharp-hoogle= walks up from the current directory looking
for a =.sln= / =.slnx=; if none is found, it falls back to a =.csproj= in the
current directory; if that also fails, it uses the running .NET runtime.

*** Project selection strategy

Currently, it can detect ~.csproj~, ~.sln~, ~.slnx~ as project file. If there're both ~.csproj~ and ~.sln~, ~.slnx~ at the same directory, ~.sln~, ~.slnx~ take priority.

1. project file specified by ~--project~ flag.
  a. When it is a project file, that will be used.
  b. When it is a directory, find valid project file under that.
  c. otherwise, result in error.
2. project files at CWD.
3. project files in parent directories.
4. If no projects are found, it falls back to use BCL only.

** Examples

Signature search:

#+begin_src sh
csharp-hoogle "IEnumerable<T> -> Func<T,bool> -> IEnumerable<T>"
#+end_src

Substring search on the method name:

#+begin_src sh
csharp-hoogle Select
#+end_src

Polymorphic signature search (lets a concrete type bind a method's generic
parameter):

#+begin_src sh
csharp-hoogle --polymorphic "int -> int -> int"
#+end_src
